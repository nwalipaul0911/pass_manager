Built for the command line

Everything you need to store and retrieve secrets without leaving your shell.

🔒

Encrypted vault

Secrets live in a single encrypted file. AES-256 via Fernet with a key derived from your master password.

🔑

Multiple secret types

Store passwords, API keys, notes, and more. Attach usernames, URLs, tags, and free-form notes.

🔍

Search & filter

Find secrets by name, username, URL, notes, or tags. Filter the list by secret type.

⚡

Script-friendly

The get command prints a field to stdout — pipe secrets into scripts and automation.

🌀

Password generator

Generate cryptographically secure passwords with configurable length and character sets.

🖥

SSH-ready

Install on a remote server, SSH in, and use it interactively. Secrets stay where you need them.

Installation

Clone, install, and create your vault in under a minute.

1

Clone & install

git clone https://github.com/nwalipaul0911/pass_manager.git
cd pass_manager
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -e .
2

Initialize your vault

pass-manager init

You'll be prompted to set a master password. There is no recovery — back it up.

3

Start adding secrets

pass-manager add github -t password -u myuser --url https://github.com
pass-manager add openai -t api_key --notes "Production key"
pass-manager list
pass-manager show github --reveal

Vault location

By default, your encrypted vault is stored at:

~/.local/share/pass_manager/vault.enc

Override with PASS_MANAGER_VAULT or the --vault flag.

Command reference

Most commands prompt for your master password. init, gen, and status do not.

Command Description
initCreate a new encrypted vault
add <name>Add a secret
list / lsList all secrets
show <name>Show details (masked by default; use --reveal)
get <name>Print a field to stdout for scripts
update <name>Update an existing secret
delete <name> / rmDelete a secret
search <query>Search by name, username, URL, notes, or tags
genGenerate a secure random password
statusShow vault path and whether it exists

Examples

Store an API key 
pass-manager add github-token -t api_key \
  --url https://github.com --tags dev,git
Generate & store a password 
pass-manager add email -t password \
  -s "$(pass-manager gen -l 32)"
Use in a script 
TOKEN=$(pass-manager get openai -f secret)
curl -H "Authorization: Bearer $TOKEN" ...
Filter by type 
pass-manager list -t api_key
pass-manager search github.com

Security model

Local-first by design. Your secrets never leave your machine unless you put them there.

Master password

Never written to disk. Used only to derive the encryption key at unlock time via PBKDF2-HMAC-SHA256 with 600,000 iterations.

Unique salt

Each vault file contains a random 16-byte salt. The same master password on different vaults produces different keys.

Restricted permissions

The vault file is created with mode 0600 where the operating system supports it.

Masked by default

show partially masks secret values. Use --reveal only when you need the full value.

No recovery

Wrong master password fails decryption. There is no hint, reset, or backdoor. Back up your master password.

Shell history

Avoid passing secrets with -s on shared systems. Prefer the interactive prompt instead.
